# CREATE UI

Read `docs/AI_RULES.md`, `docs/GAME_VISION.md`, `docs/UI_UX_RULES.md`, `docs/ART_DIRECTION.md`, `docs/TECH_ARCHITECTURE.md`, and `docs/NAMING_CONVENTIONS.md`.

Then:

- Identify whether the request belongs in the HUD, an overlay panel, the world map, or a new DOM section.
- Reuse existing panel, button, responsive, and overlay patterns where possible.
- Keep styling aligned with the current typography, color, blur, border, and spacing language.
- Preserve keyboard and close-flow behavior when modifying existing overlays.
- Verify the UI change with the project build and note any affected interaction paths.
